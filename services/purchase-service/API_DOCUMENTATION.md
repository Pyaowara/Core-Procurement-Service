# Purchase Service API Documentation

## Overview

The Purchase Service manages the complete procurement workflow including:
- **Purchase Requests (PR)**: Initiated by Employees to request items by SKU
- **Purchase Orders (PO)**: Generated by Purchase Officers from approved PRs
- **Vendor Management**: Manages vendor information
- **Message-Based Architecture**: Uses RabbitMQ for inter-service communication

## Data Models

### Purchase Request (PR)
- **pr_id**: Primary key
- **pr_number**: Unique PR number (e.g., "PR-20260321-000001")
- **requester_id**: Reference to requesting user
- **department**: Department name
- **purpose**: Purpose of the purchase request (optional)
- **status**: DRAFT, PENDING, APPROVED, REJECTED
- **workflow_id**: Link to Approval Service workflow
- **Items**: List of PR items (1 PR → many PR Items)
- **is_deleted**: Soft delete flag
- **created_at / updated_at**: Timestamps

### PR Item
- **sku**: Stock Keeping Unit (required, indexed)
- **item_name**: Name of item (fetched from Inventory Service)
- **description**: Item description (fetched from Inventory Service)
- **quantity**: Quantity requested
- **price_per_unit**: Price per unit
- **discount**: Discount amount
- **discount_unit**: % or BAHT
- **total_price**: Total price (calculated)
- **required_date**: Date item is needed (required field)

### Purchase Order (PO)
- **po_id**: Primary key
- **po_number**: Unique PO number
- **pr_id**: Reference to source PR (1 PR → many POs - multiple vendors/splits allowed)
- **vendor_id**: Reference to vendor
- **purpose**: Purpose inherited from PR
- **status**: DRAFT, SENT, COMPLETED
- **credit_day**: Payment terms (days)
- **due_date**: Expected delivery date
- **Items**: List of PO items (1 PO → many PO Items)
- **is_deleted**: Soft delete flag
- **created_at / updated_at**: Timestamps

### PO Item
- **sku**: Stock Keeping Unit (indexed)
- **item_name**: Name of item
- **description**: Item description
- **quantity**: Quantity ordered
- **price_per_unit**: Price per unit (required)
- **discount**: Discount amount
- **discount_unit**: % or BAHT
- **total_price**: Total price (required)
- **required_date**: Date item is needed

### Vendor
- **vendor_id**: Primary key
- **vendor_name**: Vendor company name
- **vendor_address**: Vendor address
- **vendor_tax_id**: Tax ID (unique)

### Data Consistency
- **GoodsReceived**: Records received items with timestamp
- No inventory snapshots are created

## Workflow

### 1. Create Purchase Request (Employee/Manager Role)
```
POST /pr
{
  "pr_number": "PR-20260321-000001",  // Optional, auto-generated if omitted
  "department": "Sales",
  "purpose": "Office supplies for quarterly budget",  // Optional
  "items": [
    {
      "sku": "SKU-001",       // Required - must exist in Inventory Service
      "quantity": 100,
      "price_per_unit": 5.50,
      "discount": 10,
      "discount_unit": "%",
      "required_date": "2026-03-15"
    }
  ]
}
```

**Process:**
1. Fetches item details (name, description) from Inventory Service by SKU
2. Creates PR with DRAFT status
3. Auto-generates PR number if not provided (format: PR-YYYYMMDD-{ID:06d})
4. Fails if SKU not found in inventory

**Response**: 201 Created
```json
{
  "message": "PR created successfully",
  "data": {
    "id": 1,
    "pr_number": "PR-20260321-000001",
    "status": "DRAFT",
    "department": "Sales",
    "purpose": "Office supplies for quarterly budget",
    "items": [
      {
        "sku": "SKU-001",
        "item_name": "Printer Paper",
        "description": "A4 paper 80gsm",
        "quantity": 100,
        "price_per_unit": 5.50,
        "discount": 10,
        "discount_unit": "%",
        "total_price": 495.00,
        "required_date": "2026-03-15T00:00:00Z"
      }
    ]
  },
  "item_validation_summary": {
    "SKU-001": {
      "item_name": "Printer Paper",
      "description": "A4 paper 80gsm",
      "quantity": 100,
      "status": "fetched from inventory"
    }
  },
  "pr_items_created_count": 1
}
```

**Error Cases:**
- `400 Bad Request` - SKU not found in inventory
- `400 Bad Request` - Required fields missing

### 3. Submit PR for Approval (Employee/Manager Role)

**Endpoint**: `POST /pr/:id/submit`

This endpoint executes a simple 3-step workflow:

#### Workflow Steps:
1. **Validate Data**
   - PR must have at least 1 item
   - All items must have `required_date` specified
   - All items must have `quantity > 0`
   - PR must be in DRAFT status

2. **Change Status**
   - PR status: DRAFT → PENDING
   - Update the `updated_at` timestamp

3. **Publish Event**
   - Publish `pr.ready.for.approval` event to RabbitMQ
   - Approval Service receives event and initiates approval workflow
   - Event includes all PR items with SKU, quantity, and pricing information

#### Success Response:
```json
{
  "message": "PR submitted successfully",
  "data": {
    "id": 1,
    "pr_number": "PR-20260321-000001",
    "status": "PENDING",
    "department": "Sales",
    "purpose": "Office supplies for quarterly budget",
    "workflow_id": "WF_1_20260305150405",
    "items": [
      {
        "sku": "SKU-001",
        "item_name": "Printer Paper",
        "description": "A4 paper 80gsm",
        "quantity": 100,
        "price_per_unit": 5.50,
        "discount": 10,
        "discount_unit": "%",
        "total_price": 495.00,
        "required_date": "2026-03-15T00:00:00Z"
      }
    ],
    "updated_at": "2026-03-05T15:04:05Z"
  },
  "workflow_id": "WF_1_20260305150405",
  "approval_event_published": true
}
```

#### Error Responses:
- **400 Bad Request** - Validation failed (missing items, no required_date, etc.)
- **404 Not Found** - PR not found
- **500 Internal Server Error** - Failed to publish event

### 4. Receive Approval Events (Auto-Subscribe)
PR status updates based on approval service events:
- **approval.completed**: PR status → APPROVED
- **approval.rejected**: PR status → REJECTED

### 5. Generate Purchase Order (PurchaseOfficer Role)
Multiple POs can be created from the same PR by specifying different vendors and items.

```
POST /po
{
  "pr_id": 1,
  "vendor_id": 1,
  "credit_day": 30,
  "due_date": "2026-04-15",
  "item_ids": [1, 2, 3]
}
```

**Parameters:**
- `pr_id` (required): ID of approved PR
- `vendor_id` (required): ID of vendor
- `credit_day` (optional): Payment terms in days
- `due_date` (required): Expected delivery date (YYYY-MM-DD)
- `item_ids` (optional): Array of PR item IDs to include in PO. If not provided, all PR items will be used. All item IDs must belong to the specified PR.

**Validation:**
- All PR item IDs must belong to the specified PR, otherwise returns 400 error
- At least 1 valid PO item must be created, otherwise PO is not created and returns 400 error

**Response:**
- Status: DRAFT
- Created with selected PR item details (SKU, quantity, price, etc.)
- Publishes: `po.created` event to RabbitMQ

**Example with all items:**
```
POST /po
{
  "pr_id": 1,
  "vendor_id": 1,
  "credit_day": 30,
  "due_date": "2026-04-15"
}
```
All items from PR will be included in the PO.

**Example with selected items:**
```
POST /po
{
  "pr_id": 1,
  "vendor_id": 1,
  "credit_day": 30,
  "due_date": "2026-04-15",
  "item_ids": [1, 3]
}
```
Only items with IDs 1 and 3 will be included in the PO (both must belong to PR ID 1).

**Error Cases:**
- `400 "PR item ID X does not belong to this PR"`: Requested item ID not found in PR
- `400 "no valid PO items to create"`: No valid items to create PO from

### 6. Update PO Status (PurchaseOfficer Role)
```
PUT /po/:id
{
  "status": "SENT"
}
```
Valid statuses: DRAFT, SENT, COMPLETED

### 7. Record Goods Reception (Manager/PurchaseOfficer Role)
```
POST /po/:id/receive
{
  "received_qty": {
    "Printer Paper": 100
  }
}
```
- Publishes: `goods.received` event to RabbitMQ
- Event consumed by Inventory Service to update stock

## Authentication

All endpoints (except `/health`) require JWT token in either:
1. Cookie: `token`
2. Header: `Authorization: Bearer <token>`

### Role-Based Access Control

| Role | Permissions |
|------|------------|
| **Employee** | Create, update, submit PRs (own records) |
| **Manager** | Create, update, submit PRs + receive goods/record reception |
| **PurchaseOfficer** | Generate POs, update PO status, manage purchase operations |
| **Executive** | (Reserved for approval workflow in future) |
| **Admin** | Full access - manage vendors, view all records |
| **AuthRequired** | View-only access to PR/PO lists and details |

## Data Consistency

### PR and PO Item Storage
- All PR and PO item details (SKU, quantity, price, discount) are stored in the database when records are created
- Item names and descriptions are fetched from Inventory Service by SKU at creation time
- No snapshots are created - the database tables themselves serve as the audit trail

### Soft Deletes
- PRs and POs use soft delete (is_deleted flag)
- Records are kept for audit trail
- Deleted records are not returned in list queries

## Error Handling

All endpoints return JSON responses with appropriate HTTP status codes:
- **200 OK**: Successful GET/PUT operation
- **201 Created**: Successful POST operation
- **400 Bad Request**: Invalid request data
- **401 Unauthorized**: Missing or invalid authentication
- **403 Forbidden**: User lacks required role
- **404 Not Found**: Resource not found
- **500 Internal Server Error**: Server error

## Integration Notes

### With Approval Service
- Sends PR data via RabbitMQ event when submitted
- Subscribes to approval decisions and updates PR status accordingly

### With Inventory Service
- Publishes `goods.received` events when items are received
- Should subscribe to inventory-related events for cross-checking

## Development Notes

The purchase service follows the same coding patterns as the inventory-service:
- Uses GORM for database operations
- Uses Gin for HTTP routing
- JWT middleware for authentication
- RabbitMQ for async event-driven communication
- Models use gorm.Model for timestamps and soft deletes
